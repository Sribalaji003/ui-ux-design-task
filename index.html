WF-01

Setup Wizard — Step 1: PM Locations

Wizard
Annotations
1 Auto-launch trigger
Wizard fires on first module access. localStorage flag inventory_setup_complete gates this. Once set, wizard is only accessible via Settings.
2 Read-only cards
Locations come from PM API call (GET /locations). No edit, add, or delete affordances. "Read-only" badge communicates intentional constraint, not a bug.
3 Progress stepper
4-step linear flow. Steps 2–4 are disabled until step 1 completes. Tab key is trapped within the active step content area.
4 Next button
Enabled immediately (read-only review). Keyboard shortcut: Alt+→ advances. Focus moves to first interactive element in Step 2.

WF-02

Setup Wizard — Step 2: Sub-Locations CRUD

Wizard
Annotations
1 + Add inline
Clicking + Add appends a blank editable row at the bottom of the table. Focus auto-moves to the Name input. Enter saves, Esc cancels.
2 Location scoping
Sub-locations are created per Location. Tabs switch context. Adding a sub-loc in OFFICE_A tab does not affect OFFICE_B.
3 Duplicate validation
On blur, if name already exists for this Location, inline error: "Name already in use for this location." Save button for that row disables.
4 Delete guard
Delete icon always visible in wizard (no balance exists yet). Post-setup, delete is guarded if balance > 0 (shows Archive instead).
5 Back navigation
Back returns to Step 1. Entered sub-locations are preserved in state (not cleared). Alt+← goes back.

WF-03

Setup Wizard — Step 3: CSV Import

Wizard
Annotations
1 Two-phase import
Items master imported first (creates catalogue). Balances imported second (seeds quantities). Balances upload only unlocks after Items import succeeds.
2 Template download
"Download Template" generates a header-only CSV with one example row. Filename: items_template.csv. Helps users match column names exactly.
3 Preview table
Shows first 5 rows after parse. Helps user confirm columns mapped correctly before full upload is committed.
4 Error handling
Row-level errors shown as: "X rows valid, Y rows errored." Error detail downloadable as error_report.csv. All errors must be resolved before Confirm is enabled (no partial import).
5 Idempotency
Re-uploading same file is safe — upsert by SKU. No duplicate rows created. Useful when fixing a subset of errors.

WF-05

Items List — Primary (Daily Count)

Primary Screen
Annotations
1 Location selector
Pill buttons. Persisted in localStorage. Switching location resets sub-location to "All" and re-fetches. ARIA role="radio" group.
2 Sub-location tabs
Cascades from Location. Tab key moves between tabs (not arrow — arrow is reserved for table row navigation). "All" tab shows aggregate, New Qty inputs disabled.
3 Search / filter
Client-side filter on SKU + Name. Debounced 200ms. Active filter shown as dismissible chip. Ctrl+F focuses search field.
4 New Qty keyboard flow
Tab moves to next row's New Qty. Enter commits that row and moves down. Shift+Tab moves up. Empty = no change (skipped on Save All).
5 Save All
Disabled when no rows are dirty or any row has a validation error. Optimistic update: Current Qty flips immediately on click; reverts on API error. Badge shows count of dirty rows.
6 Row-level save
Checkmark icon appears on hover of a dirty row. Commits only that row. For careful single-item corrections without affecting others.

WF-06

Items List — Validation Error State

Edge State
Annotations
1 Inline error timing
Error shown within 300ms of blur event. Field border turns red. Error icon + text below field. Never colour-only (accessibility).
2 Save All disabled
Save All button becomes disabled (greyed) when any field has an active error. Tooltip explains: "Fix validation errors before saving." Tab focus skips the disabled button.
3 Error types
Non-numeric: "Enter a number." Negative: "Must be 0 or greater." Over limit: "Maximum is 9,999." All use same visual pattern.

WF-07

Items List — Concurrency Conflict Modal

Edge State
Annotations
1 Trigger condition
API returns HTTP 409 when server's updated_at for a Balance row differs from the ETag sent in the Save All request. Indicates another user saved between page-load and current save.
2 Diff display
Shows three values: Original (page-load value), Theirs (what the other user saved), Mine (what I typed). User makes an informed decision.
3 Resolution actions
"Keep mine" — sends a force-write with my value. "Accept theirs" — discards my input, updates UI to their value. Both dismiss the modal. If multiple rows conflict, modal shows them as a list.
4 Non-conflicting rows
Rows without conflicts are already committed by the time the 409 is returned. Only conflicting rows need resolution.

Items List — Empty State

Edge State
Annotations
1 Two empty variants
Variant A (shown): sub-location exists but no items assigned. CTA directs admin to add items. Variant B: search returns no results — show "Clear search" CTA instead.

Empty states never say "No data." They explain the situation and offer a clear path forward. Don Norman: affordances must always point to action.

Sub-Location Management

Annotations

WF-10

Audit Log

Audit
Annotations
1 Default state
Loads last 30 days, all locations, newest first. Pagination: 50 rows per page with virtual scroll on large sets.
2 Filter cascade
Location filter populates Sub-Location dropdown. Date range: calendar picker or type YYYY-MM-DD. All filters are additive (AND logic).
3 Delta column
Shows +N or −N. Red for negative delta, green for positive. Helps managers spot anomalies at a glance without computing diffs manually.
4 CSV export
Exports current filter view. Columns: timestamp, actor, location, sublocation, sku, item_name, old_qty, new_qty, delta, source. Max 10,000 rows per export.
5 Source badge
"Manual" = daily count entry. "CSV Import" = wizard or re-import. Helps distinguish data entry from bulk uploads in audit reviews.

WF-11

Add / Edit Item Modal

Management
Annotations
1 Modal vs page
Add/Edit is a modal overlay (not a new page) to preserve the user's place in the filtered list. Focus is trapped within modal. Esc dismisses with a discard-changes confirm if the form is dirty.
2 SKU immutability
In Edit mode, SKU field is read-only (grayed, no focus). Tooltip: "SKU cannot be changed after creation." This protects audit trail integrity.
3 Live SKU check
In Add mode, SKU field triggers a debounced (500ms) uniqueness check on blur. Green check = available. Red = taken.
4 Archive vs Delete
Archive button always shown on Edit modal. Delete only shown if balance = 0 AND no transactions. Delete requires a confirmation dialog with item name typed to confirm.
